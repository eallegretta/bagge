1                        Introducción

1.1        Propósito

El objetivo del presente documento, es explicar el funcionamiento del software para el Sistema de Seguimientos de Tickets  a ser implementado en UNITEL S.A. Está orientado para el usuario final y debe servir de guía por medio de instrucciones paso a paso.

El contenido abarca todas las funcionalidades que fueron previstas y fueron contempladas durante todas las fases de su construcción.

2                        MU001  Ingreso al sistema

2.1        Autenticación al Sistema

Acceder al sistema SETI por medio de.

Ingrese su usuario y clave en los campos correspondientes, luego apriete el botón Login.

Actores

Usuario

Objetivo

Ingresar al sistema

Nombre del Caso de Uso

Autenticación al Sistema

Pre Condiciones

Que el usuario no haya ingresado al sistema.

Post Condiciones

Que el usuario haya ingresado al sistema.

 

Curso Normal (curso normal del caso de uso)

Caso Alternativo

1. El usuario ingresa a la pantalla de ingreso al sistema

 

2. El sistema solicita usuario y contraseña

 

3. El usuario ingresa usuario y contraseña y presiona en el botón Login

3.1 El usuario presiona en Recordar contraseña Ir a caso de uso Recuperar Contraseña (extiende)

4. El sistema verifica que los datos sean correctos y permite acceso al sistema

Fin caso de uso.

4.1 El sistema muestra un mensaje de error si los datos ingresados no son correctos

 

 

Resultado Esperado: El usuario ingresa al sistema
